# Exact environment that produced the versioned artifacts under results/.
#
# Why this file exists separately from pyproject.toml: the ranges declared there
# are the library's compatibility contract, exercised by the CI matrix across
# Python 3.10-3.13. They are not a reproducibility contract. NumPy freezes the
# random stream only for the legacy RandomState (NEP 19); a `Generator` may
# produce different draws after a feature release, so identical code and
# identical seeds reproduce the recorded numbers only against pinned versions.
#
# Interpreter: CPython 3.13.0
#
# Install with:
#   python3.13 -m venv .venv && . .venv/bin/activate
#   pip install -r replication/requirements-lock.txt
#   pip install -e . --no-deps
#
# Note that numpy 2.5 requires Python >= 3.12, so this environment is narrower
# than the package's declared `requires-python = ">=3.10"`. The matrix proves the
# code runs on 3.10-3.13; this lock is what reproduces the results bit-for-bit.
#
# Dataset provenance is pinned separately: the nine knapsack instances come from
# pisinger-knapsack v0.2.0, which this repository does not depend on at runtime.
# See the "Knapsack dataset provenance" section of replication/README.md.
